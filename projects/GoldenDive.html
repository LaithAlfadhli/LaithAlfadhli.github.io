Lessons Learned

The Golden Dive was the first time I was tasked to program the code for a game project intended to be made into a game or form the basis of a game. I learned how to differentiate between variables that needed to be private and which ones needed to be public so that my co-worker could edit and tune the values if required.

I also learned that balancing the game is crucial to player enjoyment. Our game was meant to put pressure on the player by having a timer in which you had to complete the game; however, we made the timer longer due to a limited number of testers winning the game. The longer time made the timer a non-issue in most testers games. I will be more careful when balancing a game to avoid making a mechanic obsolete in the future.

I learned how to work with dictionaries and prefab spawning. I learned a method for implementing checkpoints. I learned the basics of how to write a design document.
